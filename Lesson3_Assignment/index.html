Highlight #1 Versioning Control System

The versioning control system is a great and efficient way to share files. It allows you to see who changed the files, when they were changed, what exactly was changed, and then merge them into the same file. Multiple people can simultaneously work on the same file and since no file can be overwritten, you are able to go back to an older version of the project. All of the latest files are held in a common repository.

Highlight #2 Repositories

A repository is made to control history. It is created with files and directories so that you can record the file's history and track their versions and status. There are two types of repositories; local and remote. The local repository is a repository that is on a local machine of an individual user. The remote repository resides on a remote server and shared among multiple users.

Highlight #3 GIT

GIT is an open and free versioning control system specifically deigned to handle everything from very small to very large files with speed and efficiency. With GIT, every user has a copy of their repository data stored locally on their machine. Processes are able to run much faster with GIT and there is no single point of failure. Git will also not allow an unintentional overwrite of a file on a shared repository.
